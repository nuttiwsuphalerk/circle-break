# วงแตก (Circle Break) — Static Web Build

This is the **static, framework-free build** of วงแตก, meant for GitHub +
Cloudflare Pages (also works on GitHub Pages, Netlify, Vercel, or any static
host — it's just HTML/CSS/JS).

It's the same modular source as the Google Apps Script version, but a small
Node build script (`build.js`) now does what GAS's `include()` used to do —
at build time instead of on every request, which is also faster.

**This build includes Google AdSense.** The original GAS version's footer
promised "no ads, everything stays on your device" — that's no longer
accurate once ads are in, so the footer/Settings copy here has been updated
to disclose it honestly. If you don't want ads, delete the `<script>` tag
and `.ad-slot` block from `src/index.html` and you're back to the original,
ad-free, fully-local app.

## Project layout

```
build.js              Node script: stitches src/ into dist/index.html
package.json           `npm run build` → runs build.js
src/
  index.html           page shell (includes AdSense script + ad slot)
  css.html
  js.html
  components/*.html
  games/*.html
  static/
    ads.txt             REQUIRED by AdSense — must end up at your domain root
    robots.txt
dist/                  generated by `npm run build` — gitignored, don't commit
```

## 1. Build locally (optional, just to check it works)

```bash
npm run build      # or: node build.js
```

This writes `dist/index.html` plus `dist/ads.txt` and `dist/robots.txt`.
Open `dist/index.html` directly in a browser, or serve it locally:

```bash
npx serve dist          # or: python3 -m http.server -d dist 8080
```

## 2. Push to GitHub

```bash
git init
git add .
git commit -m "วงแตก — static build"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```

(`dist/` and `node_modules/` are already in `.gitignore` — Cloudflare builds
`dist/` itself on every deploy, so you never commit generated output.)

## 3. Connect Cloudflare Pages

1. Cloudflare dashboard → **Workers & Pages** → **Create application** → **Pages** → **Connect to Git**.
2. Pick your GitHub repo and authorize access.
3. Build settings:
   - **Framework preset:** None
   - **Build command:** `npm run build` (or `node build.js`)
   - **Build output directory:** `dist`
   - **Root directory:** `/` (leave default, unless you nested this project in a subfolder)
4. Click **Save and Deploy**. Cloudflare will run the build and give you a
   `*.pages.dev` URL within a minute or two.
5. (Optional) **Custom domains** tab → add your own domain, following
   Cloudflare's DNS instructions. AdSense generally wants a real domain
   rather than a `*.pages.dev` subdomain — see below.

Every future `git push` to `main` triggers a new automatic deploy.

## 4. Set up Google AdSense

1. Apply at [adsense.google.com](https://adsense.google.com) with your
   **live site URL** (your custom domain, once connected — AdSense review
   generally wants a real domain, and wants to see actual, substantial
   content already live, not a placeholder page).
2. Approval isn't guaranteed and isn't instant — Google reviews for original
   content, navigability, and policy compliance. A single-page party-game
   app with sparse content may get held up in review; having the game pages,
   this README's disclosures, and a few visits of real traffic helps.
3. Once approved, AdSense gives you a **publisher ID** (`ca-pub-XXXXXXXXXXXXXXXX`)
   and, if you create a manual ad unit, a **slot ID**. Replace the placeholders:
   - In `src/index.html`: both `ca-pub-XXXXXXXXXXXXXXXX` occurrences, and `data-ad-slot="XXXXXXXXXX"`.
   - In `src/static/ads.txt`: the `pub-XXXXXXXXXXXXXXXX` value.
4. Commit and push — Cloudflare redeploys automatically.
5. In the AdSense dashboard, check **Sites → ads.txt status** — it should
   flip to "Authorized" once Cloudflare has redeployed with your real ID
   (may take a few hours to recheck).

### About consent (EEA / UK / Switzerland visitors)

Google requires publishers serving ads to those regions to get user consent
for cookies before ads that use them load. The simplest way to handle this
without writing custom consent code: in AdSense, go to **Privacy & messaging**
and turn on a consent message (Google's own, certified CMP) — it will
auto-inject a consent banner ahead of the ad script. I haven't built a custom
consent banner into this codebase; if your audience is meaningfully EU/UK,
turn that on before you go live. (I'm not a lawyer — for anything beyond
AdSense's own consent tooling, e.g. a full privacy policy page, get that
checked by someone qualified for your jurisdiction.)

### Ad placement, by design

There's exactly **one** ad slot, positioned below all game content — never
next to a button, the spin wheel, the race track, or the teeth grid. That's
intentional: AdSense policy prohibits placement that could cause accidental
clicks or implies the ad is part of game controls. If you want more ad
slots later, keep them equally isolated from interactive elements.

## Reverting to the original ad-free GAS version

The Google Apps Script version (with `Code.gs`, no ads, original footer
copy) is unaffected by any of this — it's a separate zip/deployment. This
`circle-break-web` project is a distinct build meant specifically for the
public, ad-supported, custom-domain deployment.
